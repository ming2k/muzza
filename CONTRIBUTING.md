# Contributing to muzza

This project welcomes focused, well-tested contributions.

## Getting Started

1. Read the [Developer Setup](docs/dev/setup.md) guide.
2. Review the [Code Style](docs/dev/code-style.md) and [Testing](docs/dev/testing.md) standards.
3. Check the [Task Backlog](docs/dev/backlog.md) for `ready` items, or open an issue to propose a change.

## Pull Request Etiquette

- One logical change per PR.
- Include tests for project-model or behavior changes.
- Update docs when changing architecture, file format, or workflows.
- Ensure `meson test -C build` passes before submitting.
- Do not mix formatting churn with feature changes.

## Governance

See [Autonomous Governance](docs/dev/governance.md) for authority boundaries, merge gates, and escalation rules.
